# Test Domain Memory

## Memory 1: Basic Setup
When setting up a new project, always create a README.md first. Include:
- Project description
- Installation instructions
- Usage examples
- License information

## Memory 2: Code Organization
Organize code by feature, not by type. This makes it easier to:
- Find related files
- Understand the codebase
- Maintain and refactor

## Memory 3: Documentation Tips
Good documentation should:
1. Be written for the target audience
2. Include examples that actually work
3. Be kept up-to-date with code changes
4. Use clear, simple language

## Memory 4: Testing Strategy
Always write tests that:
- Cover edge cases
- Are independent of each other
- Run quickly
- Provide clear failure messages
